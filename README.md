# HeadAgent — 고개 제스처 음성 에이전트 (실동작 프로토타입)

휴대폰을 **가로로 세운 자세**(화면이 사용자 쪽, 내부 카메라가 왼쪽으로 90° 누운 상태)에서,
**고개를 꺾고 말하면** 그 말을 Gemini가 받아 답해주는 핸즈프리 음성 에이전트입니다.
`headmotion.js` 모듈을 입력 트리거로, Web Speech API를 음성 입출력으로,
Gemini API를 두뇌로 사용합니다.

## 동작 흐름

1. 고개를 **왼쪽 아래로 숙이면**(트리거) → 그 자세를 유지하는 동안 마이크 ON
2. 말을 하고 → 고개를 **정면으로 펴면** → 마이크 OFF, 인식 문장이 Gemini로 전송
3. Gemini 응답은 **음성(TTS)으로 전달**됩니다. 화면 표시는 데모 확인용 미러입니다.
4. "**지금부터 같이 봐줘**" 류로 말하면 **외부(후면) 카메라**가 켜지고
   **전체화면 프리뷰**가 뜹니다. 이후 질문은 카메라 프레임과 함께 Gemini로
   전달됩니다. "**카메라 꺼**" 또는 우측 상단 ✕ 로 종료.

### "왼쪽 아래로 숙이기" 트리거 방식

"왼쪽 아래"는 **아래로 숙임(pitch)** + **왼쪽으로 기울임(roll)** 두 회전이
섞인 동작입니다. 한 축만 보면 둔하게 잡혀서, 이 앱은 두 각도를 따로 추정해
**둘 중 하나라도 임계를 넘으면 트리거**합니다. 그래서 동작이 한쪽으로
치우쳐도 잘 인식됩니다. 화면 상단의 **숙임 / 좌우** 두 게이지로 각각이
얼마나 잡히는지 실시간으로 볼 수 있습니다.

## 실행 방법 — HTTPS 필수

DeviceMotion 센서·마이크·카메라는 **보안 컨텍스트(HTTPS)** 에서만 동작합니다.
`file://` 로 직접 열면 센서가 조용히 막힙니다. 아래 중 하나로 호스팅하세요.

- **가장 쉬움:** GitHub Pages, Netlify, Vercel 등에 `index.html` 업로드
- **빠른 테스트:** PC에서 `python3 -m http.server` 실행 후
  ngrok / cloudflared 같은 터널로 HTTPS 주소를 만들어 폰에서 접속
- iOS Safari / Android Chrome 모두 지원 (음성 인식은 Chrome 권장)

휴대폰에서 페이지를 연 뒤:

1. **Gemini API Key** 입력 → "API Key 연결 테스트" 로 확인
   (키는 이 브라우저 localStorage 에만 저장, 서버 전송 없음)
2. "**▶ 센서 시작**" → 모션·마이크 권한 허용
3. 휴대폰을 얼굴 옆에 가로로 들고 고개를 꺾어 말해보기

## 새 기기에서 축 보정 (중요)

휴대폰이 90° 가로로 누워 있어 **어느 자이로 축이 숙임/좌우에 잡히는지가
기기마다 다릅니다.** 동작이 이상하면 페이지 하단 "축 보정" 섹션에서:

1. 고개를 **위아래로만** 숙였다 들기 → `α/β/γ` 중 제일 크게 변하는 값이 `tiltAxis`
2. 고개를 **좌우로만** 기울이기 → 제일 크게 변하는 값이 `swipeAxis`
3. 화면 상단의 **숙임 / 좌우** 게이지 막대가 의도와 반대로 차면, 해당
   **방향 뒤집기** 스위치(숙임 / 좌우 / 왼쪽 기울임)를 켭니다
4. 보정값은 자동 저장되어 다음 접속 때 유지됩니다

기본값은 `tiltAxis=beta`, `swipeAxis=alpha` 이며, 가로 자세에서는
`tiltAxis=gamma` 가 맞는 기기도 흔합니다 — 위 절차로 확인하세요.
"고개 중심 재설정" 버튼을 누르면 현재 자세가 숙임·좌우 모두 0°가 됩니다.

## 파일

- `index.html` — 단일 파일 앱. `headmotion.js` 모듈이 내부에 인라인되어 있어
  이 파일 하나만 호스팅하면 동작합니다.
- `headmotion.js` — 원본 모듈 (참고용 별도 사본).

## 조절 가능한 것

화면 안에서 바꿀 수 있습니다: Gemini 모델(flash / flash-lite / pro),
음성 언어(한/영/일), 트리거 방향(아래로 숙임 ↔ 위로 듦),
"매 질문마다 카메라 자동 첨부" 옵션, 그리고 위의 축 보정값.
응답 음성(TTS)은 이 에이전트의 1차 출력이므로 항상 켜져 있습니다.

## 알려진 제약

- 음성 인식(Web Speech API)은 브라우저 의존적입니다. iOS Safari는
  지원이 제한적이라 인식이 끊길 수 있습니다 — Android Chrome 이 가장 안정적입니다.
- 자이로 센서가 없는 기기에서는 트리거가 동작하지 않습니다
  (`no-events` 상태로 표시됨).
- 카메라는 **외부(후면) 카메라**를 우선 요청하며(`facingMode:'environment'`),
  켜지면 전체화면 프리뷰가 뜹니다. 프레임은 가로 자세에 맞춰 90° 회전 보정 후
  전송됩니다. 기기 방향이 반대면 `index.html` 의 `#camFullVideo` CSS 회전각과
  `captureFrame()` 회전각을 함께 조정하세요.
- API 키가 클라이언트에 노출되므로, 실제 배포 시에는 키를 서버 측
  프록시 뒤에 두는 것을 권장합니다 (이 프로토타입은 데모 목적).
